<objective>
Fix two Pokédex data-integrity bugs in `roulette-container.component.ts`:

1. **SHINY-03** — `registerInPokedex` is called in `completePokemonCapture` before the shiny roulette runs, so the Pokédex entry is always written with `shiny: false`. After `setShininess(true)` sets `makeShiny()` on the team member, the Pokédex is never updated.

Purpose: Pokédex accurately reflects shiny captures and champion wins for all Pokémon including alt-forms.
Output: Two targeted one-line/block edits in the component + two new spec tests confirming the fixes.
</objective>

**Fix 0 — Prerequisite: assign `currentContextPokemon` in `completePokemonCapture`:**

Pokémon without forms skip form-selection and go directly through `completePokemonCapture`. Without this fix `currentContextPokemon` is `undefined` when `setShininess` runs, so the spread in Fix 1 produces `{ shiny: true }` with no `pokemonId` — writing to `caught['undefined']` instead of `caught['1']`.

Current code:
```typescript
private completePokemonCapture(pokemon: PokemonItem): void {
  this.trainerService.addToTeam(pokemon);
  this.registerInPokedex(pokemon);
  this.gameStateService.setNextState('check-shininess');
  this.finishCurrentState();
}
```

Replace with:
```typescript
private completePokemonCapture(pokemon: PokemonItem): void {
  this.currentContextPokemon = pokemon; // ensures setShininess can reference captured pokemon
  this.trainerService.addToTeam(pokemon);
  this.registerInPokedex(pokemon);
  this.gameStateService.setNextState('check-shininess');
  this.finishCurrentState();
}
```
