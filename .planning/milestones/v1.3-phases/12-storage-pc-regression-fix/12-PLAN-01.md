# Phase 12 Plan 01: Fix StoragePC Drag-and-Drop Write-Back

## Objective

Restore team editing via PC storage drag-and-drop by adding a write-back API to `TrainerService` and wiring it into `StoragePcComponent.drop()`.

## Root Cause

In Phase 8 (IMMUT-01), `getTeam()` was changed to return `[...this.trainerTeam]` (a copy). `StoragePcComponent` opens the modal and assigns these copies to local fields. CDK DragDrop then mutates the local copies correctly — but `updateTeam()` re-broadcasts from the service's original (unchanged) internal array, so the drag result is lost.

## Tasks

### Task 1: Add `commitTeamAndStorage()` to TrainerService

**File:** `src/app/services/trainer-service/trainer.service.ts`

Add a new public method after `updateTeam()`:

```typescript
commitTeamAndStorage(team: PokemonItem[], stored: PokemonItem[]): void {
  this.trainerTeam = [...team];
  this.storedPokemon = [...stored];
  this.trainerTeamObservable.next(this.getTeam());
}
```

This writes the drag-and-drop result back to the service's internal state and broadcasts the change.

### Task 2: Update `StoragePcComponent.drop()` to call the write-back

**File:** `src/app/trainer-team/storage-pc/storage-pc.component.ts`

Replace:
```typescript
this.trainerService.updateTeam();
```
With:
```typescript
this.trainerService.commitTeamAndStorage(this.trainerTeam, this.storedPokemon);
```

### Task 3: Add unit test for `commitTeamAndStorage()`

**File:** `src/app/services/trainer-service/trainer.service.spec.ts`

Add a test verifying that `commitTeamAndStorage()` updates the internal state and emits via `getTeamObservable()`.

### Task 4: Verify compile and tests pass

Run `npx tsc --noEmit` and `npm test -- --watch=false`.
