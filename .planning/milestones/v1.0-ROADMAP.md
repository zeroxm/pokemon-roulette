## Overview

Made Pokémon types mechanically meaningful in gym and Elite Four battles. The player's team composition now influences wheel odds — type advantage shifts win probability and surfaces a modal explaining why — turning team building into genuine strategy rather than pure luck.

## Milestone Summary

**Key Decisions:**
- `type-matchups.json` at project root (typed via `PokemonType` union, zero HTTP overhead)
- "Strong" = any of the Pokémon's types is SE against any opponent type (player-friendly, simple rule)
- Strong takes precedence over weak in label calculation
- Wheel weight is COUNT-based (extra WheelItem entries, weight 1 each)
- No setTimeout anywhere — all async handled via Angular lifecycle + RxJS

**Known Tech Debt (deferred to v1.1):**
- Double type-advantage modal on multi-slot leader encounters (BATTLE-02, E4BATTLE-02 partial)
- No VERIFICATION.md or Nyquist VALIDATION.md files for any phase
- E4 modal reuses `gym` i18n namespace

**Out of Scope (explicitly excluded):**
- Champion battle type matchups
- Rival battle type matchups
- Move-level type calculation
- Net advantage (attack vs defence combined)
