# Milestone v1.2: Theming System

**Status:** SHIPPED 2026-04-17
**Phases:** 11
**Total Plans:** 2

## Overview

Replaced the dark/light mode toggle with a named theme selector. Users can choose between three themes — Starters (dark + Pokemon tile background), Plain Dark, and Plain Light — with the preference persisted to localStorage across sessions. All 14 components that previously responded to DarkModeService were migrated to the new ThemeService.isDark$ observable.

## Milestone Summary

**Key Decisions:**
- Default theme is always starters — no conditional migration from old dark-mode key
- DarkModeService kept (not removed) — only the toggle component is replaced
- isDark$ = theme$ piped through map(t => t !== plain-light) — backward compat with all templates
- localStorage.removeItem(dark-mode) called on every setTheme() to clean legacy key
- ThemeSelectorComponent uses ngModel + (change) event instead of [selected] per plan deviation → works fine

**Issues Resolved:**
- 14 components frozen in dark-mode state on Plain Light theme (migrated to isDark$)
- Old dark-mode localStorage key not cleaned up on theme change (added removeItem call)

**Issues Deferred:**
- ThemeSelectorComponent label vertical alignment (cosmetic, user deferred to fix manually)
- DarkModeService source file not deleted (kept for DI compatibility — future cleanup)

**Technical Debt Incurred:**
- DarkModeService still injected in 14 components alongside ThemeService (unused, cosmetic)
