# Plan 02: ThemeSelectorComponent + Settings Integration + i18n

## Goal

Create the standalone `ThemeSelectorComponent` — a labeled `<select>` dropdown that reads the current theme from `ThemeService` and calls `setTheme()` on change. Wire it into the settings panel by replacing `<app-dark-mode-toggle>` with `<app-theme-selector>`. Add theme i18n keys to all 6 locale files.

This plan is **self-contained** — it creates/modifies component files, settings files, and locale JSON only. It does **not** touch `theme.service.ts` or `styles.css`. It can execute in parallel with PLAN-01.

**Prerequisite assumption:** `ThemeService` (created by PLAN-01) is available at `src/app/services/theme-service/theme.service.ts` and exports:

```typescript
export type Theme = 'starters' | 'plain-dark' | 'plain-light';

@Injectable({ providedIn: 'root' })
export class ThemeService {
  readonly theme$: Observable<Theme>;
  get currentTheme(): Theme { ... }
  setTheme(theme: Theme): void { ... }
}
```

If running PLAN-02 before PLAN-01 completes, the TypeScript compiler will flag missing imports — run PLAN-01 first, or run both in parallel if the executor environment supports it.

### Task 1: Create `ThemeSelectorComponent` (4 files)

#### 1a. `src/app/settings/theme-selector/theme-selector.component.ts`

**Action**: CREATE

```typescript
import { Component } from '@angular/core';
import { CommonModule } from '@angular/common';
import { FormsModule } from '@angular/forms';
import { TranslatePipe } from '@ngx-translate/core';
import { ThemeService, Theme } from '../../services/theme-service/theme.service';

@Component({
  selector: 'app-theme-selector',
  standalone: true,
  imports: [CommonModule, FormsModule, TranslatePipe],
  templateUrl: './theme-selector.component.html',
  styleUrl: './theme-selector.component.css'
})
export class ThemeSelectorComponent {
  constructor(public themeService: ThemeService) {}

  get currentTheme(): Theme {
    return this.themeService.currentTheme;
  }

  onThemeChange(event: Event): void {
    const select = event.target as HTMLSelectElement;
    this.themeService.setTheme(select.value as Theme);
  }
}
```

**Notes:**
- `themeService` is `public` so the template can access `currentTheme` directly if needed.
- `onThemeChange` reads `event.target` as `HTMLSelectElement` — no `[(ngModel)]` binding used, keeping it explicit and avoiding `FormsModule` two-way binding complexity with a union type.
- `FormsModule` is imported in case future revisions add `ngModel`, and for consistency with the project's form patterns.
