## v1.1 Requirements

### Layout & UI

- [x] **LAYOUT-01**: The Pokédex button and PC button row uses `justify-content-between` so they sit at opposite ends of the trainer-team panel (not `gap-2` side-by-side)
- [x] **LAYOUT-02**: The Pokédex grid shows exactly 6 columns on mobile (≤576px) and exactly 9 columns on desktop (>576px); cells remain 40px square

### Game Hook Coverage

- [x] **HOOK-01**: When a Pokémon evolves via any path (`evolvePokemon`, both branches of `evolveSecondPokemon`, or the inline `replaceForEvolution` at line 355 of roulette-container), the evolved Pokémon's ID is marked as seen in the Pokédex
- [x] **HOOK-02**: When a trade event completes (`performTrade`), the received Pokémon (`pkmnIn`) is marked as seen in the Pokédex

### Pokédex Data

- [x] **DATA-07**: A complete per-generation Pokédex data file (`pokedex-by-generation.ts`) exists as a TypeScript `Record<number, number[]>` for each of the 9 generations, listing all Pokémon IDs in consecutive national dex range order — including legendaries, mythicals, Ultra Beasts, and Gen 9 Paradox Pokémon
- [x] **DATA-08**: The Local Dex ("{Region} Dex") tab in `PokedexComponent` uses `pokedexByGeneration` from the new complete data file instead of the roulette pool `pokemonByGeneration`

### User Experience

- [x] **UX-01**: The Local Dex tab label is dynamically "{Region} Dex" (e.g. "Kanto Dex", "Paldea Dex") using `GenerationItem.region` + ngx-translate interpolation (`{{region}}`); all 6 locale i18n files updated
