<objective>
Apply two surgical CSS/HTML changes that fix the visual layout of the Pokédex feature:

1. **Button row (LAYOUT-01):** The `<div>` wrapping `<app-storage-pc>` and `<app-pokedex>` currently uses `d-flex gap-2`, which places both buttons side-by-side with only a small gap. Replacing `gap-2` with `justify-content-between` pushes them to opposite ends of the row.

2. **Grid columns (LAYOUT-02):** `.pokedex-grid` currently uses natural flex-wrap with no width constraint, producing an uncontrolled number of columns. Adding `max-width: 392px` (desktop: 9 × 40px + 8 × 4px gaps) and a `@media (max-width: 576px)` override of `max-width: 260px` (mobile: 6 × 40px + 5 × 4px gaps) enforces the intended column counts without touching cell sizes.

Purpose: Deliver the designed UI — buttons at opposite ends, fixed-column dex grid.
Output: Two modified files, 141/141 tests still passing.
</objective>

  <action>
    In `src/app/trainer-team/pokedex/pokedex.component.css`, find the existing `.pokedex-grid` rule block:

    ```css
    .pokedex-grid {
      display: flex;
      flex-wrap: wrap;
      gap: 4px;
      padding: 4px;
      /* Natural wrapping: PokedexEntryComponent cells are fixed 40px wide */
      /* ~17 cols on lg modal desktop, ~8 cols on mobile full-screen — no max-width needed */
    }
    ```

    Replace it with:

    ```css
    .pokedex-grid {
      display: flex;
      flex-wrap: wrap;
      gap: 4px;
      padding: 4px;
      /* Desktop: 9 cols × 40px cells + 8 gaps × 4px = 392px */
      max-width: 392px;
      margin: 0 auto;
    }

    @media (max-width: 576px) {
      .pokedex-grid {
        /* Mobile: 6 cols × 40px cells + 5 gaps × 4px = 260px */
        max-width: 260px;
      }
    }
    ```

    Remove the old comment lines about "~17 cols" and "no max-width needed" — they are now incorrect.

    Per LAYOUT-02: cells are already 40px fixed-width (set by `PokedexEntryComponent`). The grid `gap` is 4px. The math:
    - 9 cols: 9 × 40 + 8 × 4 = 360 + 32 = 392px
    - 6 cols: 6 × 40 + 5 × 4 = 240 + 20 = 260px

    `margin: 0 auto` centres the constrained grid within the modal.
    The breakpoint ≤576px matches Bootstrap's `sm` breakpoint, consistent with `modal-fullscreen-sm-down`.
  </action>
