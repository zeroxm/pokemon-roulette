# Plan 03: Persistent Lookup Map in PokemonService (LOOKUP-01)

**Phase:** 8 — Service Hardening
**Plan:** 03
**Requirement(s):** LOOKUP-01
**Status:** Pending

## Goal
Build `pokemonById: Map<number, PokemonItem>` once in the constructor and reuse it in both
`getPokemonById()` and `getPokemonByIdArray()`, eliminating the O(n) linear scan and the throwaway
Map allocation that currently happen on every call.

## Files Changed
- `src/app/services/pokemon-service/pokemon.service.ts` — add persistent Map field, update two methods

No test file changes — the two existing `getPokemonByIdArray` tests cover identical observable
behavior and will pass unchanged.

---

## Changes

### `src/app/services/pokemon-service/pokemon.service.ts`

#### 1. Add the persistent Map field and initialize it in the constructor

**Before (lines 11–15):**
```typescript
export class PokemonService {

  constructor(private http: HttpClient) { }

  private apiBaseUrl = 'https://pokeapi.co/api/v2';
  nationalDexPokemon = nationalDexPokemon;
```

**After:**
```typescript
export class PokemonService {

  private readonly pokemonById: Map<number, PokemonItem>;

  constructor(private http: HttpClient) {
    this.pokemonById = new Map(
      this.nationalDexPokemon.map(pokemon => [pokemon.pokemonId, pokemon])
    );
  }

  private apiBaseUrl = 'https://pokeapi.co/api/v2';
  nationalDexPokemon = nationalDexPokemon;
```

> **Note:** `nationalDexPokemon` is a class field initialized at declaration (line below the
> constructor), so the class field initializers run before the constructor body in Angular's
> compiled output — the Map is safely built from the complete array. This matches the established
> pattern seen in other services in this codebase.

---

#### 2. Replace the O(n) scan in `getPokemonById()`

**Before (lines 39–42):**
```typescript
getPokemonById(pokemonId: number): PokemonItem | undefined {
  const pokemon = this.nationalDexPokemon.find(pokemon => pokemon.pokemonId === pokemonId);
  return pokemon;
}
```

**After:**
```typescript
getPokemonById(pokemonId: number): PokemonItem | undefined {
  return this.pokemonById.get(pokemonId);
}
```

---

#### 3. Replace the throwaway Map in `getPokemonByIdArray()`

**Before (lines 44–50):**
```typescript
getPokemonByIdArray(pokemonIds: number[]): PokemonItem[] {
  const pokemonById = new Map(this.nationalDexPokemon.map(pokemon => [pokemon.pokemonId, pokemon]));

  return pokemonIds
    .map(pokemonId => pokemonById.get(pokemonId))
    .filter((pokemon): pokemon is PokemonItem => pokemon !== undefined);
}
```

**After:**
```typescript
getPokemonByIdArray(pokemonIds: number[]): PokemonItem[] {
  return pokemonIds
    .map(pokemonId => this.pokemonById.get(pokemonId))
    .filter((pokemon): pokemon is PokemonItem => pokemon !== undefined);
}
```

The local `const pokemonById` variable is deleted entirely; `this.pokemonById` is used directly.
The `.filter()` type guard is preserved verbatim.
