# Requirements — Pokédex Feature (v1)

## v1 Requirements

### Data & Persistence

- [ ] **DATA-01**: System tracks a Pokémon as "seen" when it is assigned to the player's team by the roulette (any run, any outcome, any generation)
- [ ] **DATA-02**: System tracks a Pokémon as "won" when the player defeats the Champion with it on their team
- [ ] **DATA-03**: Pokédex data (seen and won Pokémon IDs, and their cached sprite URLs) persists in localStorage — survives browser close and page refresh
- [ ] **DATA-04**: Pokédex data is NOT reset when the player starts a new game run — it is cumulative across all runs
- [ ] **DATA-05**: Fetched sprite URLs (front_default) are stored in localStorage alongside seen/won IDs so reopening the Pokédex does not re-call PokeAPI for already-known Pokémon
- [ ] **DATA-06**: PokeAPI sprite calls are deduplicated within a session via an in-memory cache (Map<pokemonId, spriteUrl>) — prevents duplicate HTTP calls if a Pokémon is seen multiple times in a session

### Visual States

- [ ] **VIS-01**: Unseen Pokémon display the `unknown.png` image (already used in the app) — no sprite loading required until a Pokémon is caught
- [ ] **VIS-02**: Seen Pokémon (assigned to team at any point) display their `front_default` sprite from PokeAPI (not `official-artwork`)
- [ ] **VIS-03**: Won Pokémon (seen + on team at Champion defeat) display a gold ★ badge in the cell corner on top of their sprite
- [ ] **VIS-04**: Each Pokédex cell shows the Pokédex number at all times (even for unseen entries)
- [ ] **VIS-05**: Pokémon name is shown once the Pokémon is seen (hidden/masked for unseen entries)
- [ ] **VIS-06**: When a Pokémon transitions from unseen to seen while the Pokédex modal is open, a CSS reveal animation plays on that cell

### Navigation & Layout

- [x] **NAV-01**: A "Pokédex" button is displayed alongside the existing `trainer.storage.access` PC button in the trainer-team area
- [x] **NAV-02**: The Pokédex modal has two tabs — "Local Dex" (current generation's Pokémon) and "National Dex" (all 1,025 Pokémon)
- [x] **NAV-03**: Each tab shows a progress counter: number of seen Pokémon out of the total for that view (e.g. "45 / 151 seen" for Local, "67 / 1025" for National)
- [x] **NAV-04**: The Pokédex modal is full-screen on mobile devices (Bootstrap `modal-fullscreen-sm-down` pattern)
- [x] **NAV-05**: The Pokédex modal respects dark mode — uses the existing dark mode theme like all other modals

## v2 Requirements (Deferred)

These are expected features deferred to the next iteration:

- Pokéball icon on "seen but not won" cells (third visual layer)
- Per-generation progress breakdown in National Dex view (e.g. "Gen 1: 45/151")
- Pokédex completion reward or notification
- Filtering or searching the Pokédex by generation, type, or name
- Shiny tracking in the Pokédex
- Full Pokédex entries (types, stats, descriptions)
- Per-run Pokédex reset option

## Out of Scope

- **Backend/server storage** — app is fully client-side (GitHub Pages static deployment)
- **Pokédex entries (types, stats, descriptions)** — v1 is catch-tracking only; full entries require significant additional data sourcing
- **Shiny Pokédex tracking** — shiny status is per-run only in the current architecture
- **Per-run Pokédex reset** — the Pokédex is explicitly global and cumulative; per-run tracking is a separate future feature
- **Filtering/search** — deferred; v1 grid is browseable by scrolling only
