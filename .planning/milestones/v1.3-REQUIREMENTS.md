## v1.3 Requirements

### Storage PC Team Editing

- [x] **PC-01**: `TrainerService` exposes a `commitTeamAndStorage(team: PokemonItem[], stored: PokemonItem[]): void` method that writes the provided arrays into the service's internal `trainerTeam` and `storedPokemon` state and immediately broadcasts the updated team via `trainerTeamObservable`.
- [x] **PC-02**: `StoragePcComponent.drop()` calls `commitTeamAndStorage()` with the locally-modified `trainerTeam` and `storedPokemon` arrays after every CDK DragDrop event, replacing the current broken `updateTeam()` call.
- [x] **PC-03**: After a drag-and-drop operation in the PC storage modal, the trainer team displayed in the main game view updates immediately without requiring a reload or any additional user action.
- [x] **PC-04**: After a drag-and-drop operation changes the team composition, the win odds displayed in any active battle roulette update reactively (all battle roulette components already subscribe to `trainerTeamObservable` — this requirement validates the end-to-end chain is working).
