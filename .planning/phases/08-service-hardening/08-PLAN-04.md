**Phase:** 8 — Service Hardening
**Plan:** 04
**Requirement(s):** STATE-01

## Goal
Replace the hardcoded 25-element state array in `initializeStates()` with a programmatic builder
driven by a `GENERATION_GAME_CONFIG` constant, and inject `GenerationService` so
`resetGameState()` produces the correct stack for whatever generation is active.

## Files Changed
- `src/app/services/game-state-service/game-state.service.ts` — add config constant, inject
  GenerationService, rewrite `initializeStates()` and update `resetGameState()` / constructor

No test file changes needed — `GameStateService` and `GenerationService` are both
`providedIn: 'root'`; Angular TestBed auto-provides both. The programmatic stack for Gen 1
(gymCount=8, eliteFourCount=4) is element-for-element identical to the current hardcoded array,
so all existing tests pass without modification.

**After:**
```typescript
private initializeStates(gymCount: number = 8, eliteFourCount: number = 4): void {
  const stack: GameState[] = ['game-finish', 'champion-battle'];

  for (let i = 0; i < eliteFourCount; i++) {
    stack.push('elite-four-battle');
  }

  stack.push('elite-four-preparation');

  for (let i = 0; i < gymCount; i++) {
    stack.push('gym-battle');
    if (i < gymCount - 1) {
      stack.push('adventure-continues');
    }
  }

  stack.push('start-adventure');
  stack.push('starter-pokemon');
  stack.push('character-select');

  this.stateStack = stack;
}
```

#### 5. Update `resetGameState()` to read the active generation

**Before (lines 85–90):**
```typescript
resetGameState(): void {
  this.initializeStates();
  this.setNextState('game-start');
  this.finishCurrentState();
  this.currentRound.next(0);
}
```

**After:**
```typescript
resetGameState(): void {
  const genId = this.generationService.getCurrentGeneration().id;
  const config = GENERATION_GAME_CONFIG[genId] ?? { gymCount: 8, eliteFourCount: 4 };
  this.initializeStates(config.gymCount, config.eliteFourCount);
  this.setNextState('game-start');
  this.finishCurrentState();
  this.currentRound.next(0);
}
```

Note: `resetGameState()` is called from `main-game.component.ts` **after** the player selects a
generation in the wheel — `GenerationService.getCurrentGeneration()` already reflects the chosen
generation at that point, so the stack is built for the correct generation.

## Notes
- No circular dependency: `GenerationService` imports only `GenerationItem`, `BehaviorSubject`,
  and `Observable`. It has no reference to `GameStateService`.
- `TrainerService` already injects both `GenerationService` and `GameStateService`; adding
  `GenerationService` as a `GameStateService` dependency does not create a cycle.
- The default parameter values `(gymCount: number = 8, eliteFourCount: number = 4)` are a
  belt-and-suspenders fallback — the callers always pass explicit values, but the defaults preserve
  backward compatibility if `initializeStates()` is ever called without arguments.
