# Plan 02: Dead-code Removal + Immutable Accessors in TrainerService (CLEAN-01, IMMUT-01)

**Phase:** 8 — Service Hardening
**Plan:** 02
**Requirement(s):** CLEAN-01, IMMUT-01
**Status:** Pending

## Goal
Remove the no-op self-assignment `pokemonIn = pokemonIn` from `replaceForEvolution()`, and make
`getTeam()` / `getStored()` return shallow copies so external callers cannot mutate the internal
arrays by reference.

## Files Changed
- `src/app/services/trainer-service/trainer.service.ts` — delete one line (CLEAN-01); change two return statements (IMMUT-01)

No test file changes required — existing tests access `service.trainerTeam` directly, not via
`getTeam()`, so they are unaffected by the copy semantics change.

---

## Changes

### `src/app/services/trainer-service/trainer.service.ts`

#### CLEAN-01 — Delete the self-assignment at line 164

**Before (lines 162–165):**
```typescript
replaceForEvolution(pokemonOut: PokemonItem, pokemonIn: PokemonItem): void {
  pokemonIn.shiny = pokemonOut.shiny;
  pokemonIn = pokemonIn;             // ← DELETE THIS LINE
  this.loadPokemonSpriteIfMissing(pokemonIn);
```

**After:**
```typescript
replaceForEvolution(pokemonOut: PokemonItem, pokemonIn: PokemonItem): void {
  pokemonIn.shiny = pokemonOut.shiny;
  this.loadPokemonSpriteIfMissing(pokemonIn);
```

The line above (`pokemonIn.shiny = pokemonOut.shiny;`) is correct shiny propagation — **do not touch it**.

---

#### IMMUT-01 — Return spread copies from the public getters

**`getTeam()` — before (line 114–116):**
```typescript
getTeam(): PokemonItem[] {
  return this.trainerTeam;
}
```

**`getTeam()` — after:**
```typescript
getTeam(): PokemonItem[] {
  return [...this.trainerTeam];
}
```

---

**`getStored()` — before (line 122–124):**
```typescript
getStored(): PokemonItem[] {
  return this.storedPokemon;
}
```

**`getStored()` — after:**
```typescript
getStored(): PokemonItem[] {
  return [...this.storedPokemon];
}
```

---

## Notes
- **Do NOT change** `removeFromTeam()` — it calls `this.trainerTeam.indexOf(pokemon)` on the
  **live** internal array before splicing. That is correct internal access and must stay.
- **Do NOT change** `updateTeam()` — it passes `this.trainerTeam` directly to `next()`. That is
  also correct internal access (action method, not a public getter).
- All internal code that mutates `this.trainerTeam` or `this.storedPokemon` directly (push, splice,
  assignment) is inside TrainerService itself and is correct — leave it unchanged.
- Shallow copy is sufficient here: elements are `PokemonItem` objects whose identity is used for
  `indexOf` lookups on the internal arrays, not on the returned copies.
