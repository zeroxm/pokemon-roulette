  <action>
Open `src/styles.css` and append the following CSS block at the end of the file. Do NOT modify any existing rules.

```css
/* MOB-01: Force fullscreen Pokédex modal on small screens.
   Bootstrap's modal-fullscreen-sm-down class uses max-width: 575.98px but
   ng-bootstrap portal rendering and viewport meta tag interactions on mobile
   can prevent it from applying. This override ensures reliability. */
@media (max-width: 576px) {
  .modal-fullscreen-sm-down .modal-dialog {
    width: 100vw !important;
    max-width: 100vw !important;
    height: 100vh !important;
    margin: 0 !important;
  }
  .modal-fullscreen-sm-down .modal-content {
    height: 100vh !important;
    border-radius: 0 !important;
  }
}
```

This rule is placed in `src/styles.css` (global stylesheet) — NOT in a component's CSS — because ng-bootstrap renders modals in a portal outside the component's view encapsulation scope. A component-scoped CSS file would not reach the modal DOM.

Run `ng test --no-watch --browsers=ChromeHeadless` to confirm no regressions.
  </action>

**Step 1 — Add `@keyframes glow-pulse`** before the `.pokedex-cell.won` rule (insert before line 11):

```css
@keyframes glow-pulse {
  0%, 100% {
    box-shadow: 0 0 4px 1px rgba(255, 215, 0, 0.5);
  }
  50% {
    box-shadow: 0 0 10px 3px rgba(255, 215, 0, 0.95);
  }
}
```

**Step 2 — Replace the `.pokedex-cell.won` rule** (currently lines 11–14):

REMOVE:
```css
.pokedex-cell.won {
  border-color: gold !important;
  box-shadow: 0 0 6px rgba(255, 215, 0, 0.7) !important;
}
```

REPLACE WITH:
```css
.pokedex-cell.won {
  border-color: gold !important;
  animation: glow-pulse 2s ease-in-out infinite;
}
```

The static `box-shadow` line is removed entirely — the animation's keyframes define both low (0%/100%) and high (50%) glow states, making a continuously pulsing effect. `border-color: gold !important` remains so the gold border is always visible regardless of animation state.

Run `ng test --no-watch --browsers=ChromeHeadless` to confirm no regressions.
