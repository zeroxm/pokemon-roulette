# Plan 01: Fix Subscription Cleanup (SUB-01, SUB-02)

**Phase:** 7 — Component Hygiene
**Plan:** 01
**Requirement(s):** SUB-01, SUB-02
**Status:** Pending

## Goal
Add `takeUntilDestroyed()` to both `RestartGameButtonComponent` and `SettingsButtonComponent` so their `wheelSpinningObserver` subscriptions are automatically cleaned up when the component is destroyed, preventing memory leaks.

## Files Changed
- `src/app/restart-game-button/restart-game-button.component.ts` — import `takeUntilDestroyed`, pipe it before `.subscribe()`
- `src/app/settings-button/settings-button.component.ts` — same pattern

## Reference Pattern
`src/app/main-game/coffee-button/coffee-button.component.ts` already uses this exact pattern (line 2 + line 21). Copy it verbatim.

## Changes

constructor(private modalService: NgbModal,
            private gameStateService: GameStateService

  this.gameStateService.wheelSpinningObserver.pipe(takeUntilDestroyed()).subscribe(state => {

  });

            private gameStateService: GameStateService,

## Notes
- `takeUntilDestroyed()` is called **without arguments** — it picks up `DestroyRef` automatically via injection context when called inside a constructor. No explicit `DestroyRef` injection is needed.
- No changes to templates, providers, or `imports[]` arrays are required.
