# Plan 01: ThemeService + Global CSS

## Goal

Create the `ThemeService` Angular service that manages three named visual themes (`starters`, `plain-dark`, `plain-light`), persists the user's selection in `localStorage`, and applies the correct CSS class to `document.body` on every load. Also add the three theme CSS class rules to the global stylesheet.

This plan is **self-contained** вЂ” it creates `theme.service.ts` and modifies `styles.css` only. No component files, no HTML templates, no i18n JSON files are touched. It can execute in parallel with PLAN-02.

## Requirements Covered

- **THEME-01** вЂ” ThemeService with `theme$`, `currentTheme`, `setTheme()`, localStorage persistence, body class management
- **THEME-02** вЂ” Global CSS rules for all three theme classes
- **THEME-05** вЂ” Migration: default to `starters` when `pokemon-roulette-theme` key is absent from localStorage

## Context: Existing Renderer2 Pattern

The following is the exact pattern used by `DarkModeService` (the canonical reference). `ThemeService` follows this same pattern.

```typescript
// src/app/services/dark-mode-service/dark-mode.service.ts (key lines)
import { Injectable, Renderer2, RendererFactory2 } from '@angular/core';

constructor(private rendererFactory: RendererFactory2, ...) {
  this.renderer = this.rendererFactory.createRenderer(null, null);
  // ^^ Renderer2 cannot be injected directly in services вЂ” use RendererFactory2
}

// Body class manipulation:
this.renderer.removeClass(element, someClass);
this.renderer.addClass(element, someClass);
```

**Why `RendererFactory2`?** Angular's `Renderer2` is a component-level abstraction. Services cannot inject it directly. `RendererFactory2.createRenderer(null, null)` returns a platform renderer safe to use in service context.

## Tasks

### Task 1: Create `ThemeService`

**File**: `src/app/services/theme-service/theme.service.ts`
**Action**: CREATE (new file вЂ” create the directory too)

Create this file with the **exact** content below:

```typescript
import { Injectable, Renderer2, RendererFactory2 } from '@angular/core';
import { BehaviorSubject, distinctUntilChanged, Observable } from 'rxjs';

export type Theme = 'starters' | 'plain-dark' | 'plain-light';

const STORAGE_KEY = 'pokemon-roulette-theme';
const DEFAULT_THEME: Theme = 'starters';

const ALL_THEME_CLASSES: string[] = [
  'theme-starters',
  'theme-plain-dark',
  'theme-plain-light',
  // Legacy classes left by DarkModeService вЂ” remove them too
  'dark-mode',
  'light-mode',
];

@Injectable({
  providedIn: 'root'
})
export class ThemeService {
  private readonly renderer: Renderer2;
  private readonly _theme$: BehaviorSubject<Theme>;

  /** Observable that emits the current theme. Only fires on distinct changes. */
  readonly theme$: Observable<Theme>;

  constructor(rendererFactory: RendererFactory2) {
    this.renderer = rendererFactory.createRenderer(null, null);

    const stored = localStorage.getItem(STORAGE_KEY) as Theme | null;
    const initial: Theme =
      stored === 'starters' || stored === 'plain-dark' || stored === 'plain-light'
        ? stored
        : DEFAULT_THEME;

    this._theme$ = new BehaviorSubject<Theme>(initial);
    this.theme$ = this._theme$.asObservable().pipe(distinctUntilChanged());

    // Apply immediately вЂ” also handles THEME-05 migration (writes default to storage)
    this.setTheme(initial);
  }

  /** Synchronous read of the current theme value. */
  get currentTheme(): Theme {
    return this._theme$.value;
  }

  /**
   * Apply a theme:
   * 1. Removes ALL old theme + legacy dark-mode/light-mode classes from body
   * 2. Adds `theme-${theme}` to body
   * 3. Persists selection to localStorage
   * 4. Emits new value on theme$
   */
  setTheme(theme: Theme): void {
    const body = document.body;

    // Remove all known theme classes (including legacy ones from DarkModeService)
    ALL_THEME_CLASSES.forEach(cls => this.renderer.removeClass(body, cls));

    // Apply the selected theme class
    this.renderer.addClass(body, `theme-${theme}`);

    // Persist
    localStorage.setItem(STORAGE_KEY, theme);

    // Emit (BehaviorSubject + distinctUntilChanged handles dedup)
    this._theme$.next(theme);
  }
}
```

**Why this implementation:**
- The constructor validates the stored value against the known union type вЂ” any unrecognised string (including `null`) falls back to `'starters'`, satisfying THEME-05 migration.
- `setTheme` is called during construction so body classes are correct from the first render.
- `ALL_THEME_CLASSES` includes `dark-mode` and `light-mode` so sessions with old DarkModeService state are cleaned up automatically.
- No `afterNextRender()` needed here вЂ” unlike DarkModeService's preloading class removal, body class application is synchronous and safe at construction time.
